# Security Policy

## Supported Versions

The following table outlines the versions of BuyThisDomain that currently receive security updates:

| Version | Supported          |
| ------- | ------------------ |
| 5.1.x   | :white_check_mark: |
| 5.0.x   | :x:                |
| 4.0.x   | :white_check_mark: |
| < 4.0   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability in BuyThisDomain, please follow these steps:

1. **Report Privately**: Do not disclose vulnerabilities publicly until they are resolved.
2. **Create a GitHub Issue**: Open a new issue in the repository and label it as `security`.
3. **Provide Details**: Include a detailed description of the vulnerability, steps to reproduce, and any potential impact.
4. **Response Time**: The security team aims to acknowledge reports within **48 hours** and provide updates within **7 days**.
5. **Resolution Process**: If the vulnerability is confirmed, we will work on a fix and may request additional details from the reporter.
6. **Public Disclosure**: After a fix is deployed, we will disclose the vulnerability in a security advisory.

We appreciate responsible disclosure and will credit contributors who report valid security issues, unless anonymity is requested.

## Security Updates

Security patches are released as needed for supported versions. Users are strongly encouraged to keep their installations up to date to benefit from the latest security fixes.

## Contact

For any security concerns, please create an issue in this repository with the `security` label.
